Compute the medium time power calulations according to [Kim] .

Parameters

  • M (int) ‚Äì the temporal integration factor.

Returns

medium time power values.

Return type

(numpy.ndarray)

Note

\[\tilde{Q}[m, l]=\frac{1}{2 M+1} \sum_{m^{\prime}=m-M}^{m+M} P[m^{\prime}]\]

where \(\tilde{Q}\) is the medium time power, \(P\) is the power, \(m\) is the frame index, \(l\) is te channel index and \(M\) is the temporal integration factor.

Note

\[\begin{split}\tilde{Q}_{out}[m, l]=\left\{\begin{array}{l} \lambda_{a}\tilde{Q}_{out}[m-1, l]+(1-\lambda_{a})\tilde{Q}_{in}[m, l], & \text{if } \tilde{Q}_{in}[m, l]\geq \tilde{Q}_{out}[m-1, l] \\ \lambda_{b}\tilde{Q}_{out}[m-1, l]+(1-\lambda_{b})\tilde{Q}_{in}[m, l], & \text{if } \tilde{Q}_{in}[m, l]<\tilde{Q}_{out}[m-1, l] \end{array}\right.\end{split}\]

where \(\tilde{Q}_{in}\) and \(\tilde{Q}_{out}\) are arbitrary input and output, \(\lambda_{a}\) and \(\lambda_{b}\) are filter related parameters, \(m\) is the frame index, \(l\) is the channel index.

spafe.features.pncc.asymmetric_noise_suppression_with_temporal_masking(Q_tilde: numpy.ndarray, threshold: float = 0) → numpy.ndarray[source]#

Apply asymmetric noise suppression with temporal masking according to [Kim].

Parameters

  • Q_tilde (numpy.ndarray) ‚Äì array representing the ‚Äúmedium-time power‚Äù.

  • threshold (float) ‚Äì threshold for the half wave rectifier.

Returns

(numpy.ndarray) array after asymmetric noise sup and temporal masking.

Note

  • 2.1 Apply asymmetric lowpass filtering.
    \[\tilde{Q}_{l e}[m, l]=\mathcal{A} \mathcal{F}_{0.999,0.5}[\tilde{Q}[m, l]]\]
  • 2.2 Substract from the input medium-time power.
    \[\tilde{Q}[m, l] - \tilde{Q}_{l e}[m, l]\]

  • 2.3 Pass through an ideal half wave linear rectifier.

  • 2.4 Re-apply asymmetric lowpass filtering.

  • 2.5 Apply temporal masking.

  • 2.6 Switch excitation.

  • The previous steps can be summarised in the following graph from [Kim].

spafe.features.pncc.pncc(sig: numpy.ndarray, fs: int = 16000, num_ceps: int = 13, pre_emph: bool = True, pre_emph_coeff: float = 0.97, power=2, window: Optional[spafe.utils.preprocessing.SlidingWindow] = None, nfilts: int = 24, nfft: int = 512, low_freq: Optional[float] = None, high_freq: Optional[float] = None, scale: Literal['ascendant', 'descendant', 'constant'] = 'constant', dct_type: int = 2, lifter: Optional[int] = None, normalize: Optional[Literal['mvn', 'ms', 'vn', 'mn']] = None, fbanks: Optional[numpy.ndarray] = None, conversion_approach: Literal['Glasberg'] = 'Glasberg') → numpy.ndarray[source]#

Compute the Power-Normalized Cepstral Coefficients (PNCCs) from an audio signal, based on [Kim] [Nakamura] .

Parameters

  • sig (numpy.ndarray) ‚Äì a mono audio signal (Nx1) from which to compute features.

  • fs (int) ‚Äì the sampling frequency of the signal we are working with. (Default is 16000).

  • num_ceps (int) ‚Äì number of cepstra to return. (Default is 13).

  • pre_emph (bool) ‚Äì apply pre-emphasis if 1. (Default is True).

  • pre_emph_coeff (float) ‚Äì pre-emphasis filter coefÔ¨Åcient. (Default is 0.97).

  • power (int) ‚Äì power value to use . (Default is 2).

  • window (SlidingWindow) ‚Äì sliding window object. (Default is None).

  • nfilts (int) ‚Äì the number of filters in the filter bank. (Default is 40).

  • nfft (int) ‚Äì number of FFT points. (Default is 512).

  • low_freq (float) ‚Äì lowest band edge of mel filters (Hz). (Default is 0).

  • high_freq (float) ‚Äì highest band edge of mel filters (Hz). (Default is samplerate/2).

  • scale (str) ‚Äì monotonicity behavior of the filter banks. (Default is ‚Äúconstant‚Äù).

  • dct_type (int) ‚Äì type of DCT used. (Default is 2).

  • lifter (int) ‚Äì apply liftering if specified. (Default is None).

  • normalize (str) ‚Äì apply normalization if approach specified. (Default is None).

  • fbanks (numpy.ndarray) ‚Äì filter bank matrix. (Default is None).

  • conversion_approach (str) ‚Äì erb scale conversion approach. (Default is ‚ÄúGlasberg‚Äù).

Returns

2d array of PNCC features (num_frames x num_ceps)

Return type

(numpy.ndarray)

Tip

  • scale : can take the following options [‚Äúconstant‚Äù, ‚Äúascendant‚Äù, ‚Äúdescendant‚Äù].

  • dct : can take the following options [1, 2, 3, 4].

  • normalize : can take the following options [‚Äúmvn‚Äù, ‚Äúms‚Äù, ‚Äúvn‚Äù, ‚Äúmn‚Äù].

  • conversion_approach : can take the following options [‚ÄúGlasberg‚Äù]. Note that the use of different options than the default can lead to unexpected behavior/issues.

Examples
from scipy.io.wavfile import read
from spafe.features.pncc import pncc
from spafe.utils.preprocessing import SlidingWindow
from spafe.utils.vis import show_features

# read audio
fpath = "../../../tests/data/test.wav"
fs, sig = read(fpath)

# compute pnccs
pnccs  = pncc(sig,
              fs=fs,
              pre_emph=0,
              pre_emph_coeff=0.97,
              window=SlidingWindow(0.03, 0.015, "hamming"),
              nfilts=128,
              nfft=1024,
              low_freq=0,
              high_freq=fs/2,
              lifter=0.7,
              normalize="mvn")

# visualize features
show_features(pnccs, "Power-Normalized Cepstral Coefficients", "PNCC Index", "Frame Index")
